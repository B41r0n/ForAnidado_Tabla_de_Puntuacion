# Contribuyendo a ForAnidado

Gracias por tu interés en contribuir a este proyecto educativo. Aquí hay algunas directrices para hacerlo.

## 📋 Cómo Contribuir

### 1. Fork el repositorio
```bash
# En GitHub, haz clic en "Fork"
```

### 2. Clona tu fork
```bash
git clone https://github.com/tu-usuario/ForAnidado.git
cd ForAnidado
```

### 3. Crea una rama para tu característica
```bash
git checkout -b feature/nombre-de-tu-feature
```

### 4. Realiza tus cambios
- Asegúrate de que el código compile sin errores
- Sigue el estilo de código existente
- Comenta cambios complejos
- Prueba tu código antes de hacer push

### 5. Commit y Push
```bash
git add .
git commit -m "descripción clara de los cambios"
git push origin feature/nombre-de-tu-feature
```

### 6. Abre un Pull Request
- Describe qué cambios hiciste
- Explica por qué son necesarios
- Incluye ejemplos si es relevante

## 🎯 Tipos de Contribuciones Bienvenidas

- 🐛 **Bug fixes:** Correcciones de errores
- 📚 **Documentación:** Mejoras al README o comentarios en el código
- ✨ **Ejercicios nuevos:** Variaciones educativas del programa
- 🌐 **Traducciones:** A otros idiomas
- 💡 **Mejoras:** Optimizaciones o claridades en el código

## ✅ Requisitos para Pull Requests

- [ ] El código compila sin errores (MSVC, MinGW o Clang)
- [ ] El programa se ejecuta correctamente
- [ ] La documentación está actualizada si aplica
- [ ] Los cambios mantienen el propósito educativo
- [ ] Sin archivos de compilación (.exe, .obj, etc.)

## 🚫 No Permitido

- Cambios que rompan la compatibilidad sin una buena razón
- Código que no compila
- Archivos de compilación o de caché
- Contenido no relacionado con el objetivo educativo

## 📧 Contacto

Si tienes preguntas, abre un "Issue" en GitHub o contacta directamente.

¡Gracias por ayudar a mejorar este proyecto educativo! 🙌
